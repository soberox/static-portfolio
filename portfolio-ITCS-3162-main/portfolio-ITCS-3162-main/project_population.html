The US population

Regression is a staple of predictive analytics. Linear regression is a process where we fit the best line where we base our prediction of the data, then we find the R^2 which tells us how close these predictions where to the actual results, and finaly we find the P-value to deternine if there is significant evidence that these two datasets are coorelated. Perfect for predicting the out come out of a continous dataset. I have chosen the US population as a topic where I can explore the predictive techniques used in linear regressional models. In Kraggle I found a currennt dataset of US population form 1790-2024. I chose this dataset for its simplicity, as it only has the required column to make this for this project. I'm going to use linear regression to predict the population of the states, and the entire country. I hope that with this practices, not only would I apply regression, but to also make informative prediction of future US population trends. These trends are vital for city planning, policy making, and state funding. With this project, I hope that I can provide accurate information to these officials, and ensure smart future planning.

before we can apply linear regression to this dataset, I first need to train my data. My target would be the population, then I converted all the states into numerical values, and finaly I concatenated the states and year to into X. Now with a trained dataset, we fit it into a linear regression model! We find the mean squared error, which is similar to R^2, in which they help us tell how off our predictions are from real data. The root mean squared error is about 1.6 million people off, but considering that the latest US population is about 339 miillion people this gives us a less than 0.5% error of total population. This model is 99.95% accurate, this suggest that future predictions are going to be nearly as accurate.

This code predicts the total U.S. population for future years by summing individual state-level predictions. It begins by creating a DataFrame containing future years (20242035) and identifying all state columns used in the models training data. For each state, it constructs a future input dataset where that state is marked as active and combines it with the future year values to match the models input structure. The trained regression model then predicts each states population for those years. These predictions are collected and summed across all states to estimate the total U.S. population for each future year. Finally, the results are stored in a DataFrame alongside the actual historical U.S. population totals for comparison and visualization.

Y = updated["population"]
dummy_col = bk.get_dummies(updated["state"])
X = bk.concat([dummy_col, updated["year"]], axis=1)
X_train, X_test, y_train, y_test = train_test_split(X, Y, test_size = 0.2, random_state = 42) 

model = LinearRegression()
model.fit(X_train, y_train)
y_pred = model.predict(X_test)
print(y_pred)
mse = mean_squared_error(y_test, y_pred)
rmse = np.sqrt(mse)
print("Mean Squared Error:", mse)
print("Root Mean Squared Error:", rmse)
lin = bk.Series(model.coef_, index=X_train.columns)

future_years = bk.DataFrame({'year': range(2024, 2035)})
state_columns = bk.get_dummies(updated["state"]).columns
future_predictions = []
for state in state_columns:
    future_state = bk.DataFrame(0, index=future_years.index, columns=state_columns)
    future_state[state] = 1
    X_future = bk.concat([future_state, future_years["year"]], axis=1)
    X_future = X_future[X_train.columns]
    y_future_pred = model.predict(X_future)
    future_predictions.append(y_future_pred)
total_us_pred = np.sum(future_predictions, axis=0)
actual_total = updated.groupby("year")["population"].sum().reset_index()
pred_lin = bk.DataFrame({
    "year": future_years["year"],
    "predicted_population": total_us_pred
})
    
plt.figure(figsize=(10,6))

Polynomail regression is a regession model that is similar to linear regression, but it is better tailrored for non-linear functions like a population. It does so by fitting the data into a non-linear function where the degree of the polynomial denoted by the user. For the size of this data set I chose to go with a degree of 3, because that is the maximum that my computer can handle (lol). Comapred to typical linear regression, the line should be curved. So I fitted my trained data into the model, and Evaluated by getting its root mean squared error. The rmse is was about 209,569, much lower than linear regression. This mean that this model is off about 209,569 out of 339 million people, thus having a error rate of 0.0617%. This model is about x10 more accurate than the previous model. Plotting the model, you can even see a slight disjoint compared to the linear model. This model is better than the previous one, but I want to see if there is another model that might predict better
